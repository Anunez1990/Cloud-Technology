Virtualization vs Containerization

October 29th, 2020 by Aayush Kumar

  • This page will be divided into two main sections explaining what Virtualization and Containerization are and how they compare.

  • 1. Virtualization

    What is Virtualization?

    Use of a single physical machine/ hardware to run multiple virtual machines within it. (It’s like having one computer, and having many different computers inside of it which are using the hardware of the main machine.)

    For each virtual machine, there is an allocated amount of CPU, RAM and storage, and the total amount of it being used cannot exceed the amount which is available on the physical hardware/main computer.

    Benifits:

    1. With virtualization, there is better use of the hardware resources, which gives us power saving as we don’t need to have as much power because there are fewer machines being powered/physical machines. We can consolidate multiple pieces of physical hardware into fewer pieces of hardware(s)
    2. We can save virtual machines as files, it can be saved /backed up and have them in multiple locations which is a great way to recover files. It is also flexible as virtual machines can be moved between different data centers.
    3. The flexibility of this allows for researching operating systems and creating sandboxes where things can be tested without having to have a physical unit to do it.

    2. Containerization

    What is Containerization?

    It is a type of operating system where applications are run in isolated user space called containers. Each container has its own system, like Linux or Windows, and under each container, multiple applications are run. The container shares the kernel of the host OS with other containers, therefore there is only one OS to maintain

    Containerization enables you to deploy multiple applications using the same operating system on a single virtual machine or server

    Benifits:

    1. Containers are usually small in size and can easily limit their memory and CPU usage
    2. Since containers are small in size, they are fast to launch as opposed to virtual machines where the entire operating system needs to be deployed.
